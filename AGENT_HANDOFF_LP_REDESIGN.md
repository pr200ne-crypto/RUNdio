**学び:**
- コードで描いた SVG イラストは品質が低すぎて使えない
- 常時動くマイクロアニメーション (float, pulse, bounce) はUXとして不快
- ビジュアルの質を上げるには **実写ストック写真** が最も効果的

**ラウンド 3 — 写真ベースに方針転換**
| パス | コンセプト | 判定 |
|------|-----------|------|
| `/lp-b4` | Unsplash 実写写真 + 落ち着いたフェードイン | **承認** |

## 2. 採用デザイン: `/lp-b4`

### ファイル
- **実装**: `demo-implementation/app/lp-b4/page.tsx`

### デザイン方針
- **カラー**: ブルー (`blue-600`) & ホワイト基調、明るく爽やか
- **カードデザインは使わない** (ユーザーの明確な指示)
- **写真**: Unsplash のストック写真をフルに活用
- **アニメーション**: スクロール時の FadeIn のみ (1回きり、0.6s)。常時動くアニメーションは **禁止**
- **フォント**: Noto Sans JP (既存設定を継続)

### ページ構成
1. **Nav** — 固定ヘッダー、ブルーのロゴ + 「はじめる」ボタン
2. **Hero** — フルブリード写真 + グラデーションオーバーレイ + 白テキスト + CTA
3. **Feature 1: Plan** — 写真(左) + テキスト(右)
4. **Feature 2: Run** — テキスト(左) + 写真(右) — bg-slate-50
5. **Feature 3: Review** — 写真(左) + テキスト(右)
6. **Stats** — 3カラム数字 (3ステップ / ∞パターン / 0円) — bg-slate-50
7. **CTA** — フルブリード写真 + ブルーオーバーレイ + 白テキスト
8. **Auth** — ログイン/登録フォーム (Supabase Auth)
9. **Footer**

## 5. 技術メモ

- **認証**: Supabase Auth (`@supabase/ssr`)。NextAuth は `auth.ts` / `app/api/auth/[...nextauth]/route.ts` に残っているが未使用。将来的に削除してよい
- **dev サーバー**: `npm run dev` (ポート 3040)
- **Tailwind**: v4 インストール済みだが `tailwind.config.js` は v3 形式。動作に問題なし
- **framer-motion**: LP で `FadeIn` コンポーネント (スクロール連動フェードイン) に使用。それ以外の常時アニメーションはユーザーが明確に嫌がるため **使用禁止**
