## 3. UX 原則

### 重視する体験

1. **走前**：条件検索までの導線を**3タップ以内**を目安に短く。 [仮]  
2. **走中**：**ロック画面／常時表示**でも状態が分かる。 [仮]  
3. **音声**：**自然な話し言葉**、**長すぎる独白を避ける**。

### 避けたい UX

1. 走行中の**細かいフォーム入力の強要**。  
2. **通知の乱発**によるイヤホン体験の破壊。

## 4. 画面構成（たたき台）

### 主要画面一覧

| 画面名 | 目的 | 主要要素 |
|--------|------|----------|
| ホーム | 今日のラン開始 | 開始、履歴、設定 |
| 計画（ルート・検索） | ルートとチェックポイント決定 | 地図、条件、候補リスト |
| 走行中 | 状態確認・最小操作 | 大きな指標、一時停止・終了 |
| 完了 | 振り返り・次回提案 | サマリー、提案カード |
| 設定 | 距離/時間、音声、連携 | トグル、選択リスト |

### 画面遷移図（テキスト）

```
[ホーム] → [計画] → [走行中] → [完了]
    ↓         ↓
 [設定]    [設定]
```
