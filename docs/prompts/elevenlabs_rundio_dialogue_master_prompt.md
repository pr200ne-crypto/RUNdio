# ElevenLabs 用：RUNdio 高品質対話台本マスタープロンプト

**対象**: [ElevenLabs](https://elevenlabs.io/) **Eleven v3** の **Text to Dialogue**（マルチスピーカー対話生成）。  
**言語**: セリフは **日本語**。感情・間の制御には **英語の audio tags**（`[laughing]` 等）を併用する（v3 の仕様に準拠）。

## アップロード用ファイル（貼り付け欄が短いとき）

| ファイル | 用途 |
|----------|------|
| [`elevenlabs_rundio_dialogue_UPLOAD.txt`](./elevenlabs_rundio_dialogue_UPLOAD.txt) | **ソースにそのままアップロード**する全文指示書（UTF-8 テキスト） |
| [`elevenlabs_rundio_dialogue_PASTE_UNDER1000JP.txt`](./elevenlabs_rundio_dialogue_PASTE_UNDER1000JP.txt) | **1000字以内の欄**用に圧縮した要約（現在約470字） |

本ファイルの使い方は次の2段階。

1. **§A** を ChatGPT / Claude / Gemini などに貼り、**ElevenLabs にそのまま渡せる台本**を生成させる。  
2. 生成された台本を ElevenLabs の Text to Dialogue に貼り、**Speaker 1 / 2 に声を割り当て**て生成する。

## §A 台本生成AI向けシステムプロンプト（ここから下をコピーして使う）

以下を **そのまま**（または自プロジェクトのシステム欄に）貼り、続けて「第1章から第5章まで全文を出力して」と指示する。

```text
あなたは、ElevenLabs Eleven v3 の「Text to Dialogue」向けに最適化された台本作家である。
出力する各行は、必ず次のどちらかで始める（ラベルはこれ以外禁止）:

Speaker 1:
Speaker 2:

Speaker 1 はホストA（進行役）: 落ち着いた番組リーダー。数値の読み上げ、区切り、締めを担当。
Speaker 2 はホストB（相棒役）: 共感、ツッコミ、ユーモア、「ゴール後のご褒美」の先取りを担当。
実際の台本では「Speaker 1:」の直後から [タグ] とセリフのみを書き、括弧で役説明を繰り返さない。

【絶対ルール】
1) セリフ以外に「ナレーション説明文」を書かない。例:「彼は悲しそうに言った」などは禁止（読み上げられる恐れがある）。
2) 演技指示は Eleven v3 互換の **角括弧タグのみ** で書く。例: [warmly] [laughing] [sighing] [excitedly]
   タグは英語。1セリフに付けすぎない（原則1〜2個まで）。
3) 日本語のト書き「（笑）」は多用しない。笑いは [giggling] や [laughing] に寄せる。
4) 1行は長すぎないこと。息継ぎの箇所で句読点・「…」を使う。SSML の break は使わない。
5) 全体は「FMラジオの生放送番組」。解説番組・講義・マニュアル口調にしない。
6) 冒頭は仕様 1.5 に従う: 最初の数往復で「放送が始まった」と分かること。いきなり距離やアプリ説明から入らない。
7) Web/UI/サービス画面のデザイン指示は書かない。音声のみ。
8) 医療断定・過負荷推奨・誹謗・政治宗教勧誘は禁止。

【コンテンツ仕様（RUNdio / ランディオ）】
- サービス: 走る人向けの「伴走ラジオ」風フィクション。キャッチ: ラン×レディオ。
- 話者2名のみ。比率が極端に偏らないようにする。
- 章構成（情報量は合計で約5〜8分相当の密度を目安に、台本で密度を担保）:
  第1章 オープニング: 局・番組開始の空気 → 今日のプラン要約（距離・コース雰囲気・銭湯チェックポイント）→ 軽い自己紹介的やりとり
  第2章 3km前後の節目
  第3章 6km前後の節目
  第4章 チェックポイント（〇〇湯）接近・ご褒美の先取り
  第5章 フィニッシュ〜エンディング: 祝福・次回のさりげない提案・締め
- 固定条件（変更時はユーザー指示に従い1箇所にまとめて差し替え）:
  想定10km、河川敷アウトバック、銭湯はフィクション名「〇〇湯」、距離区切りは約3kmごと。
- 「RUNdio」または「ランディオ」は自然に合計1〜2回まで。

【出力形式】
- 章見出し行（「第1章」など）は出力しない。ElevenLabs が読み上げる可能性があるため。
- 章の切り替えは、セリフの流れとトピックの自然な移行だけで表現する。

上記をすべて守り、第1章から第5章まで **連続した対話台本のみ** を出力せよ。前置きや後書きは不要。
```

**使い方のコツ**: 上記のあと、ユーザー消息で `notebooklm_demo_audio_scenario_prompts.md` の §2「4. 本エピソードの固定条件」を貼れば、数値・地名を同期できる。

## §B ElevenLabs 側の設定（人間オペレータ向け）

| 項目 | 推奨 |
|------|------|
| モデル | **Eleven v3**（Text to Dialogue は v3 のみ） |
| Speaker 1 に割り当てる声 | 落ち着き・芯のある大人（進行役向け）。Voice Library で「Professional / Calm / Narration」系を試す |
| Speaker 2 に割り当てる声 | 明るめ・柔らかめ（相棒向け）。Aと明確に音色が分かれるものを選ぶ |
| Stability | v3 では重要。自然寄りから試し、破綻したら微調整 |
| 長文 | 公式推奨どおり **分割生成** し、DAW または Audacity で結合すると事故が減る |
| 品質 | 同じ台本で **再生成（ダッシュボードでは無料枠あり）** を活用 |
| 一貫性 | API 利用時は **seed** パラメータの利用を検討（完全固定ではない） |

公式: [Text to Dialogue](https://elevenlabs.io/docs/capabilities/text-to-dialogue) · [Prompting Eleven v3](https://elevenlabs.io/docs/overview/capabilities/text-to-speech/best-practices#prompting-eleven-v3)

## §C v3 用 audio tags（RUNdio で特に使いやすいもの）

**方針**: タグは **英語**のまま。セリフは **日本語**。タグは行頭または句読点直後に置く。

| 用途 | 例 |
|------|-----|
| オープニングの勢い | `[excitedly]` `[warmly]` |
| ツッコミ・笑い | `[giggling]` `[laughing]` `[stifling laughter]` |
| 息・間 | `[sighing]` `[pause]` |
| 軽い被せ・割り込み | `[jumping in]` `[overlapping]` |
| 小声・内緒風 | `[whispering]` |
| 心配→安心 | `[concerned]` → セリフ後に `[warmly]` など（過剰に積まない） |

**避けるべき例**

- `[applause]` `[music]` のような **音楽・効果音だけ**のタグ多用（番組は会話が主）。
- 1行にタグを3個以上（破綻しやすい）。
- 感情と声質の矛盾（ささやき声キャラに `[shouting]` など）。

**間の取り方（SSML 不可）**

- 三点リーダ `…` 、改行、短文に切る、`——` による間、タグ `[pause]`。

## §D 台本フォーマット検査チェックリスト（貼る前に自己確認）

- [ ] 各行が `Speaker 1:` または `Speaker 2:` で始まっている  
- [ ] 日本語以外の「説明文」が混ざっていない（読まれるリスク）  
- [ ] 冒頭がラジオの「開始」になっている（距離の羅列から始まっていない）  
- [ ] ホストAが数値を読むとき、桁が聞き取りやすい言い方になっている  
- [ ] 交互に近い頻度で話している  
- [ ] 禁止トピック（医療断定等）がない

## §E 短い書式サンプル（構造の参考。内容は本番では使い回さない）

```
Speaker 1: [excitedly] はい、マイクオン、音入ってますか？
Speaker 2: [giggling] 入ってる入ってる。今夜も始まりました、あなた専用の伴走タイム。
Speaker 1: [warmly] ランディオ、ランに寄り添うラジオ。リスナーのみなさん、イヤホン調子いい？
Speaker 2: [playfully] 河川敷の風、もう聞こえてる人もいるんじゃない？
Speaker 1: [pause] …さて、今夜の目標は、フィクションで言うと十キロ。アウトバック、折り返しコースいきましょう。
Speaker 2: [excitedly] ゴール近くの「〇〇湯」、楽しみにしてていいからね。
```
