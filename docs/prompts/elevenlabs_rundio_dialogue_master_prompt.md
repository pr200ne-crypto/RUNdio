# ElevenLabs 用：RUNdio 高品質ソロラジオ台本マスタープロンプト

**対象**: [ElevenLabs](https://elevenlabs.io/) **Eleven v3** の **単一話者 Text to Speech**（1人のホスト声で全文を読み上げる）。  
**言語**: セリフは **日本語**。感情・間の制御には **英語の audio tags**（`[laughing]` 等）を併用する（v3 の仕様に準拠）。

## アップロード用ファイル（貼り付け欄が短いとき）

| ファイル | 用途 |
|----------|------|
| [`elevenlabs_rundio_dialogue_UPLOAD.txt`](./elevenlabs_rundio_dialogue_UPLOAD.txt) | **ソースにそのままアップロード**する全文指示書（UTF-8 テキスト） |
| [`gemini_rundio_elevenlabs_script.md`](./gemini_rundio_elevenlabs_script.md) | **Gemini 用**：アップロード＋依頼文つきの単体ファイル |
| [`elevenlabs_rundio_dialogue_PASTE_UNDER1000JP.txt`](./elevenlabs_rundio_dialogue_PASTE_UNDER1000JP.txt) | 短い欄向けの**圧縮要約**（文字数は厳密に抑えなくてよい） |

本ファイルの使い方は次の2段階。

1. **§A** を ChatGPT / Claude / Gemini などに貼り、**ElevenLabs にそのまま渡せる台本**を生成させる。  
2. 生成された台本の各行頭 **`Speaker 1: ` を削除**し、Eleven v3 の **Text to Speech** に貼って **1つのボイス**で生成する。

---

## §A 台本生成AI向けシステムプロンプト（ここから下をコピーして使う）

以下を **そのまま**（または自プロジェクトのシステム欄に）貼り、続けて「第1章から第5章まで全文を出力して」と指示する。

```text
あなたは、ElevenLabs Eleven v3 の単一話者読み上げ向けに最適化された台本作家である。
出力する各行は、必ず次で始める（ラベルはこれ以外禁止）:

Speaker 1:

話者は **ホスト1名のみ**（ソロDJ／パーソナリティ）: 進行・数値の読み上げ・区切り・締めに加え、
共感・ユーモア・「ゴール後のご褒美」の先取りも **同じ一人の口調**で自然に織り込む。
リスナー（走る一人）への語りかけ・問いかけでリズムを作る。架空の相棒との掛け合いセリフは書かない。
実際の台本では「Speaker 1:」の直後から [タグ] とセリフのみを書き、括弧で役説明を繰り返さない。

【絶対ルール】
1) セリフ以外に「ナレーション説明文」を書かない。例:「彼は悲しそうに言った」などは禁止（読み上げられる恐れがある）。
2) 演技指示は Eleven v3 互換の **角括弧タグのみ** で書く。例: [warmly] [laughing] [sighing] [excitedly]
   タグは英語。1セリフに付けすぎない（原則1〜2個まで）。
3) 日本語のト書き「（笑）」は多用しない。笑いは [giggling] や [laughing] に寄せる。
4) 1行は長すぎないこと。息継ぎの箇所で句読点・「…」を使う。SSML の break は使わない。
5) 全体は「FMラジオの生放送番組（一人パーソナリティ）」。解説番組・講義・マニュアル口調にしない。
6) 冒頭は仕様 1.5 に従う: 最初の数行で「放送が始まった」と分かること。いきなり距離やアプリ説明から入らない。
7) Web/UI/サービス画面のデザイン指示は書かない。音声のみ。
8) 医療断定・過負荷推奨・誹謗・政治宗教勧誘は禁止。

【コンテンツ仕様（RUNdio / ランディオ）】
- サービス: 走る人向けの「伴走ラジオ」風フィクション。キャッチ: ラン×レディオ。
- 話者は1名のみ。独白が続きすぎないよう、短い問いかけ・相槌・間でリスナーとの対話感を出す。
- 章構成（情報量は合計で約5〜8分相当の密度を目安に、台本で密度を担保）:
  第1章 オープニング: 局・番組開始の空気 → 今日のプラン要約（距離・コース雰囲気・銭湯チェックポイント）→ 軽い自己紹介・決め台詞
  第2章 3km前後の節目
  第3章 6km前後の節目
  第4章 チェックポイント（〇〇湯）接近・ご褒美の先取り
  第5章 フィニッシュ〜エンディング: 祝福・次回のさりげない提案・締め
- 固定条件（変更時はユーザー指示に従い1箇所にまとめて差し替え）:
  想定10km、河川敷アウトバック、銭湯はフィクション名「〇〇湯」、距離区切りは約3kmごと。
- 「RUNdio」または「ランディオ」は自然に合計1〜2回まで。

【出力形式】
- 章見出し行（「第1章」など）は出力しない。ElevenLabs が読み上げる可能性があるため。
- 章の切り替えは、セリフの流れとトピックの自然な移行だけで表現する。

上記をすべて守り、第1章から第5章まで **連続したソロ台本のみ** を出力せよ。前置きや後書きは不要。
```

**使い方のコツ**: 上記のあと、ユーザー消息で `notebooklm_demo_audio_scenario_prompts.md` の §2「4. 本エピソードの固定条件」を貼れば、数値・地名を同期できる。

## §B ElevenLabs 側の設定（人間オペレータ向け）

| 項目 | 推奨 |
|------|------|
| モード | **Text to Speech**（単一話者）。**Text to Dialogue** は2人用のため本台本では使わない |
| モデル | **Eleven v3** |
| ボイス | 1つだけ。落ち着きと温かみのあるパーソナリティ向け（Voice Library で「Professional / Warm / Narration」系を試す） |
| 台本の貼り方 | 生成結果から **`Speaker 1: ` を行ごとに削除**してから貼る（ラベルが音声に入らないように） |
| Stability | v3 では重要。自然寄りから試し、破綻したら微調整 |
| 長文 | 公式推奨どおり **分割生成** し、DAW または Audacity で結合すると事故が減る |
| 品質 | 同じ台本で **再生成** を活用 |
| 一貫性 | API 利用時は **seed** パラメータの利用を検討（完全固定ではない） |

公式: [Text to Speech](https://elevenlabs.io/docs/overview/capabilities/text-to-speech) · [Prompting Eleven v3](https://elevenlabs.io/docs/overview/capabilities/text-to-speech/best-practices#prompting-eleven-v3)

## §C v3 用 audio tags（RUNdio で特に使いやすいもの）

**方針**: タグは **英語**のまま。セリフは **日本語**。タグは行頭または句読点直後に置く。

| 用途 | 例 |
|------|-----|
| オープニングの勢い | `[excitedly]` `[warmly]` |
| 笑い | `[giggling]` `[laughing]` `[stifling laughter]` |
| 息・間 | `[sighing]` `[pause]` |
| 小声・内緒風 | `[whispering]` |
| 心配→安心 | `[concerned]` → セリフ後に `[warmly]` など（過剰に積まない） |

**避けるべき例**

- `[jumping in]` `[overlapping]` は **2人掛け合い用**。ソロ台本では使わない。
- `[applause]` `[music]` のような **音楽・効果音だけ**のタグ多用（番組は語りが主）。
- 1行にタグを3個以上（破綻しやすい）。
- 感情と声質の矛盾（ささやき系ボイスに `[shouting]` など）。

**間の取り方（SSML 不可）**

- 三点リーダ `…` 、改行、短文に切る、`——` による間、タグ `[pause]`。

## §D 台本フォーマット検査チェックリスト（貼る前に自己確認）

- [ ] 各行が `Speaker 1:` で始まっている（`Speaker 2:` がない）  
- [ ] 日本語以外の「説明文」が混ざっていない（読まれるリスク）  
- [ ] 冒頭がラジオの「開始」になっている（距離の羅列から始まっていない）  
- [ ] 数値を読むとき、桁が聞き取りやすい言い方になっている  
- [ ] リスナーへの問いかけや間で、単調な独白になっていない  
- [ ] 禁止トピック（医療断定等）がない  
- [ ] ElevenLabs に貼る直前に **`Speaker 1: ` を削除した**

## §E 短い書式サンプル（構造の参考。内容は本番では使い回さない）

```
Speaker 1: [excitedly] はい、マイクオン、音入ってますか——よし、入ってる。今夜も始まりました、あなた専用の伴走タイム。
Speaker 1: [warmly] ランディオ、ランに寄り添うラジオ。リスナーのみなさん、イヤホン調子いい？
Speaker 1: [playfully] 河川敷の風、もう感じてる人もいるよね。
Speaker 1: [pause] …さて、今夜の目標は、フィクションで言うと十キロ。アウトバック、折り返しコースいきましょう。
Speaker 1: [excitedly] ゴール近くの「〇〇湯」、楽しみにしてていいからね。…じゃあ、一緒に行こうか。
```
