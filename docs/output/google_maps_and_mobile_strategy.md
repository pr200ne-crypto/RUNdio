# Google Maps API 導入計画とコスト試算 (RUNdio)

## 1. Google Maps API の費用体系
Google Maps Platform は、毎月 **$200 分の無料クレジット** が提供されます。
個人開発や小規模なデモであれば、この無料枠内で収まることがほとんどです。

### 主な利用機能と単価 (1,000回あたり)
| 機能 | 用途 | 単価 (1,000回) | 無料枠内での上限 |
| :--- | :--- | :--- | :--- |
| **Maps JavaScript API** | 地図の表示 (Dynamic Maps) | $7.00 | 約 28,500 回/月 |
| **Places API (Nearby Search)** | ルート付近の施設検索 | $32.00 | 約 6,250 回/月 |
| **Routes API** | ルートの計算・描画 | $5.00 | 約 40,000 回/月 |

## 2. RUNdio での費用シミュレーション (月間)
1人のユーザーが毎日1回ランニング（計画〜実行）を行うと仮定：
- 地図表示: 3回 (計画時、走行中、完了後) × 30日 = 90回
- 施設検索: 1回 (計画時) × 30日 = 30回
- **合計コスト: 約 $1.59 / 月** → **無料枠 ($200) で十分カバー可能**

## 3. 導入時の注意点
- **クレジットカード登録**: 無料枠内であっても、Google Cloud Console での請求先アカウント設定とカード登録が必須です。
- **APIキーの制限**: 悪用を防ぐため、HTTPリファラー（ドメイン制限）を必ず設定してください。
- **クォータ設定**: 予期せぬ課金を防ぐため、1日あたりのリクエスト上限を設定することを推奨します。

# スマートフォンアプリとしての見せ方 (PCブラウザデモ)

## 1. モバイルフレーム UI
PCブラウザで表示する際、画面中央にスマートフォンの筐体（ベゼル）を模したフレームを配置し、その中でアプリを動作させます。これにより、審査員やユーザーに「これはモバイルアプリである」という視覚的コンテキストを強く与えます。

## 2. タッチ操作のシミュレーション
- マウスカーソルを「タッチポイント（丸いドット）」に変更。
- スワイプ操作の有効化。

## 3. LP (Landing Page) の役割
LP は「PCで見ているユーザー」を「アプリのダウンロード」や「Web版の体験」へ誘導する入り口として機能させます。
- **Hero**: アプリのコンセプトを伝えるビジュアル。
- **Features**: ラジオ体験、ルート検索などの紹介。
- **CTA**: 「今すぐ体験する（Webデモへ）」ボタン。
