# 開発計画 (RUNdio)

## 1. 開発ロードマップ

### Phase 1: MVP開発 (デモデイ向け) - **完了**
- **目標**: 計画〜走行〜完了の一連の体験をWeb上でデモ可能にする。
- **主要タスク**:
  - Next.js プロジェクトのセットアップ。
  - 画面フローの実装。
  - モックデータによるルート・POI検索。
  - 疑似走行ロジックとWeb Speech APIによる音声通知。
  - Supabase Auth による認証。
  - モバイルフレームUIの導入。

### Phase 2: 本番環境への移行 (将来)
- **目標**: 実走可能なスマートフォンアプリとしてのリリース。
- **主要タスク**:
  - React Native (Expo) への移行。
  - Google Maps API の導入。
  - 高品質なTTS（ElevenLabs等）の導入。
  - バックグラウンドGPSトラッキング。
  - 実際のPOI検索APIの実装。

## 3. リスク管理
| リスク内容 | 影響度 | 発生確率 | 対策 |
| :--- | :--- | :--- | :--- |
| ブラウザの音声自動再生制限 | 高 | 中 | ユーザー操作（開始ボタン）をトリガーに再生を開始。 |
| APIキーの露出 | 中 | 低 | 環境変数の適切な管理と、APIキーのドメイン制限。 |
| デモ時のネットワーク不安定 | 中 | 中 | モックデータ（JSON）を優先し、外部依存を最小限に。 |
