## 2. 走前（プランニング）

### 今日のランを計画する

**目的：** ルートとゴールのイメージを決め、走行中の音声が参照する「物語」を用意する。

**フロー（たたき台）：**

1. 「今日のランを始める」を選ぶ。  
2. **目標距離**または**目標時間**（またはフリーを選択）を入力。 [仮]  
3. **ルートを作成または選択**する（自動生成、過去ルート、編集等 — 詳細未定）。  
4. **ルート上の条件検索**で、例として「このルートの前方〇km付近の**入浴施設**」など**カテゴリ条件**を指定し、候補から**ゴール／チェックポイント**を選ぶ。 [仮：UI は未定]  
5. （任意）**カスタムメッセージ**や**ご褒美の一言**を紐づける。 [仮]  
6. 確認画面で**音声プラン**（大まかな区切り）を表示し、**スタート**へ。

**成功条件：** ルートと主要チェックポイントが確定し、走行セッションが開始できる。

---

## 3. 走中（メインフロー）

### ランディオ（パーソナル音声）付きラン

**目的：** 数値・励まし・番組的な流れで伴走し、他オーディオ（ラジオ等）と**共存**する。

**フロー：**

1. **セッション開始**。位置・距離（ルート上）・時間をトラッキング。  
2. **オープニング**音声（短い）。  
3. **中盤**：設定に応じた頻度で、**距離または時間**の区切りごとに**数値＋励まし**（言い換えプール）。  
4. **チェックポイント接近**：ルート上で**残り距離**が閾値を切ったら**ご褒美の先取り**等の一言。  
5. **終盤／ゴール手前**：テンションを上げる**クロージングに向けたトーク**。  
6. 外部オーディオ再生中は**ダッキング**等で**被せる**（仕様詳細は別途）。 [仮]

**成功条件：** ユーザーが中断・終了操作で**セッションを完了**できる。
