# RUNdio プロジェクト成果物サマリー

## 1. プロジェクト概要
**RUNdio（ラン×レディオ）** は、「ランニング×ラジオ」をコンセプトにした、一人で走るランナーのためのパーソナルラジオ・アプリです。
走る前のプランニングから、走行中の音声実況、走後の振り返りまでを一体の体験として提供します。

## 2. 成果物一覧

### 企画・要件定義 (`docs/input/`, `docs/output/`)
- **ビジネス要件**: 市場課題、ターゲット、収益モデルの定義。
- **ユーザー要件**: ペルソナ設定とユーザーストーリー。
- **システム要件 (v0.2)**: Webデモ・低コスト運用を前提とした要件定義。
- **詳細要件定義書**: Mermaid図（遷移図、ER図）を含む実装ガイド。

### 詳細設計 (`docs/design/`)
- **システムアーキテクチャ**: Next.js + Supabase + Vercel の構成図。
- **データベース設計**: テーブル定義とRLS（行単位セキュリティ）ポリシー。
- **API仕様書**: 走行セッション保存等のエンドポイント定義。
- **UI/UX設計**: デザインコンセプト、カラーパレット、画面遷移。
- **開発計画**: MVP開発タスクと将来のロードマップ。

### 実装 (`demo-implementation/`)
- **Next.js 15 (App Router)**: モダンなWebフレームワークによる高速なUI。
- **Supabase Auth (SSR)**: 認証済みユーザーのみが利用可能なセキュアな設計。
- **Mobile Frame UI**: PCブラウザ上でスマホアプリの体験を再現する外枠。
- **疑似走行エンジン**: タイマーと連動した距離進行と、Web Speech APIによる音声実況。
- **ランディングページ (LP)**: サービス紹介とサインアップ機能。

## 3. デモの確認方法
1. `demo-implementation` ディレクトリで `npm install` を実行。
2. `.env.local` に Supabase の認証情報を設定。
3. `npm run dev` を起動し、`http://localhost:3040/lp` にアクセス。

## 4. 今後の展望
- **ネイティブアプリ化**: React Native (Expo) によるバックグラウンドGPS対応。
- **高品質音声**: ElevenLabs API 等による、より人間らしいラジオ実況の導入。
- **リアルタイム施設検索**: Google Maps API との連携による、ルート沿いの動的なPOI提案。
